# NORKA R35 — 3D Scroll Story

A Vite + React + strict TypeScript automotive storytelling experience using the real uploaded R35 GLB. The WebGL canvas is fixed full-screen while HTML sections scroll above it. One GSAP master timeline controls camera position, target, FOV, and section copy, so upward scrolling evaluates the exact same path in reverse.

The project is configured for **Yarn Classic 1.22.22 only**. `yarn.lock` is the single dependency lockfile.

## Requirements

- Node.js 20.19+ or 22.12+
- Yarn 1.22.22
- A browser with WebGL2 and hardware acceleration
- Non-commercial use because the supplied model is licensed CC BY-NC-SA 4.0

## Setup

```bash
yarn install --frozen-lockfile
yarn dev
```

Open the local URL printed by Vite.

## Production

```bash
yarn typecheck
yarn build
yarn preview
```

The production output is `dist/`. `vercel.json` is included; on Vercel use the Vite preset, build command `yarn build`, and output directory `dist`.

## Runtime assets

- `public/models/norka-r35-original.glb` — untouched reference copy of the uploaded GLB.
- `public/models/norka-r35-desktop.glb` — quality-first web model with 4096px hero maps, KTX2 GPU textures, and 16-bit normalized surface vectors.
- `public/models/norka-r35-mobile.glb` — balanced mobile model with 2048px ETC1S paint, 1024px carbon/glass, and semantic KTX2 compression.
- `public/models/norka-r35-mobile-low.glb` — constrained mobile model with 1024px hero maps, 512px secondary maps, and KTX2 compression.
- `public/models/norka-r35-mobile-fallback.glb` — mobile compatibility model with 512px hero maps, 256px secondary maps, standard PNG textures, and no Basis worker.
- `public/models/norka-r35-fallback.glb` — unchanged desktop compatibility model with 1024px references, 512px secondary maps, and standard PNG textures.
- `public/textures/` — the supplied complete source pack. The optimizer maps the named paint, carbon, glass, seams, and interior decal assets to their existing UV/material slots; split `@channels=G/B` files are source channels, not extra runtime textures.
- Three.js bundles the matching Basis Universal transcoder revision with production. `public/basis/` remains a checked-in source copy, but is not used by the current runtime loader.
- `public/hdr/automotive-studio.hdr` and `public/hdr/automotive-studio-mobile.hdr` — local procedural 1024 × 512 desktop and 512 × 256 mobile environments generated by `scripts/generate-studio-hdr.py`.

Each generated web GLB is self-contained and embeds every car texture once. The `inspect:model:*` commands report file size, topology, KTX2 mode, texels, and model-only GPU estimates in bytes and MiB; dedicated commands cover `mobile`, `mobile-low`, `mobile-fallback`, `fallback`, and `original`. Run `yarn validate:model:mobile` to verify mobile texture caps, render vertices, material/alpha preservation, bounds, instancing, and primitive count. The visible footer attribution is read from `asset.extras` at runtime, with a defensive fallback only when metadata is absent; it also credits the vecarz material presentation.

Without an override, a mobile renderer exposing no ASTC, ETC, BC, S3TC, or PVRTC target receives `mobile-fallback`. With GPU compression available, a normal phone or coarse-pointer tablet selects balanced `mobile`; a device reporting no more than 2 GB of memory, no more than 2 logical CPU cores, or a GPU texture-size ceiling of 4096px or lower selects `mobile-low`. Desktop keeps the existing `desktop`/`fallback` behavior. The initial tier is locked for the lifetime of the Canvas, so address-bar, viewport, and orientation resizes cannot load a second GLB over the first. `VITE_MODEL_VARIANT=original`, `desktop`, `mobile`, `mobile-low`, `mobile-fallback`, or `fallback` is honored only by the development server for controlled comparisons; production always keeps the capability guardrails.

## Scroll and camera architecture

`src/hooks/useScrollStory.ts` creates a single master ScrollTrigger in normal-motion mode. Its scrubbed timeline is the only writer to the story rig. Camera position uses two non-overlapping legs per section to form the orbit/raise/descend/pull-back path. Target and FOV span the complete section transition. Reverse scroll therefore evaluates the same timeline backward, with no separate camera animation on `onLeaveBack`.

`src/three/CameraRig.tsx` copies the mutable rig into the Three.js camera in `useFrame`; React state is not updated on every scroll frame. GSAP and ScrollTrigger explicitly invalidate the demand-driven canvas only when camera or material state changes. The projection matrix is updated only when FOV changes. Section text uses opacity and transforms.

For `prefers-reduced-motion`, the camera switches between static desktop/compact shots and copy uses short opacity changes.

## Explore mode

The opening Explore CTA disables story ScrollTriggers without resetting their visual state, locks the page at its current scroll offset, eases to the opening Explore composition, then enables OrbitControls. Pan is disabled, zoom is bounded, and polar limits prevent moving below the floor. Drag/touch, wheel/pinch, arrow keys, and `+`/`-` are supported. Exit disables controls, restores the opening story shot, restores the same scroll offset, and re-enables ScrollTriggers without jumping to another section.

## Development camera HUD

In development, press **D** to toggle the HUD. It shows camera position, target, FOV, current section, and master progress. **Copy shot** copies a valid `cameraShots.ts` object entry. The module is behind `import.meta.env.DEV` and is excluded from production output.

To show ScrollTrigger markers during development:

```bash
VITE_SCROLL_MARKERS=true yarn dev
```

## Camera tuning workflow

1. Run `yarn dev` and wait for the real GLB to reach 100%.
2. Press **D** to show the camera HUD.
3. Scroll to a section and inspect the actual composition.
4. Adjust the matching entry inside `desktopShots` or `mobileShots`; use `landscapeShots` only for the short-landscape override in `src/three/cameraShots.ts`.
5. Use **Copy shot** to capture position, target, and FOV.
6. Test both down-scroll and up-scroll. Target must continue tweening for the full section transition; do not introduce independent camera tweens.
7. Test portrait framing separately. Mobile shots are authored independently, not obtained by multiplying desktop coordinates.

## Model color restoration and compression

The original GLB is never overwritten. The downloaded model contains the complete geometry, UV sets, nodes, and 64 material slots, but the linked vecarz edition applies its dark carbon, black cabin, wheel, glass, and PBR settings inside the Sketchfab viewer. Those overrides are not stored in the downloadable GLB, which is why the untreated model renders large exterior and interior areas white or gray.

At runtime, `src/three/materialAdjustments.ts` restores the linked edition by exact material name. It applies the recovered linear color factors, metalness, roughness, opacity, clearcoat, alpha cutouts, UV channels, and adaptive 4×/8× anisotropic sampling; it reuses maps embedded in the selected GLB and disables the three white emissive overrides that were not enabled in the reference presentation. Geometry, proportions, material slots, and camera normalization are unchanged.

The optimizer embeds the checked-in paint, carbon albedo/normal, glass, seams, and interior decal albedo/normal maps into their named material slots. The runtime therefore does not download or decode a second copy of those textures. The separate ground AO source is intentionally not attached to the car because the scene provides a one-frame contact shadow.

The pipeline never simplifies, welds, or removes model geometry. Mobile variants instance repeated opaque meshes and join the remaining static opaque hierarchy, while preserving transparent draw boundaries, bounds, materials, and the original 1,267,536 render vertices. Position, normal, tangent, and in-range UV data use conservative 16-bit storage; normal/tangent angular error is effectively sub-pixel while their GPU bytes and bandwidth are halved. Constant textures are folded into material factors without removing UV channels needed by runtime restoration.

Desktop color/normal textures use quality-level-3 UASTC with baked mipmaps and Zstd; linear metallic/roughness/specular masks use high-quality ETC1S. The balanced mobile tier keeps paint at 2048px but encodes it as high-quality ETC1S, while carbon and glass are 1024px; other color/normal detail remains UASTC where appropriate. `mobile-low` caps hero maps at 1024px and secondary maps at 512px using KTX2. `mobile-fallback` caps them at 512px/256px and uses standard PNG, so it never starts Basis. The desktop compatibility output remains at 1024px/512px PNG. Color-space classification comes from the glTF material slot, never PNG metadata; decal opacity is moved from normal alpha into base color before unused normal alpha is stripped.

Optimizer dependencies are pinned in `devDependencies`:

```bash
yarn install --frozen-lockfile
yarn optimize:model
```

Default outputs:

- `public/models/norka-r35-desktop.glb`
- `public/models/norka-r35-mobile.glb`
- `public/models/norka-r35-mobile-low.glb`
- `public/models/norka-r35-mobile-fallback.glb`
- `public/models/norka-r35-fallback.glb`

Use `yarn optimize:model:desktop`, `yarn optimize:model:mobile`, `yarn optimize:model:mobile-low`, `yarn optimize:model:mobile-fallback`, or `yarn optimize:model:fallback` to rebuild one variant only. The corresponding `yarn inspect:model:*` commands inspect each output, and `yarn validate:model:mobile` checks all three mobile variants against the original. High-quality KTX2 authoring is intentionally slow and prints per-texture progress. Runtime uses the KTX2Loader shipped with the installed Three.js revision, limits transcode concurrency to one worker, and terminates that worker as soon as the selected GLTF has resolved.

The checked-in HDR files are ready to use. Their generator uses only the Python 3 standard library:

```bash
python3 scripts/generate-studio-hdr.py
```

## Performance and resilience

Normal mobile is capped at DPR 2 and `mobile-low` at DPR 1.5, with quarter-step mobile DPR quantization and 3.0/1.75-megapixel budgets to avoid render-target churn as browser chrome resizes. Mobile always disables MSAA, uses a one-frame 256px contact shadow, and loads the 512 × 256 HDR. Desktop keeps its existing DPR/MSAA guards, 4.0-megapixel budget, default 512px contact shadow, and byte-identical 1024 × 512 HDR; oversized desktop canvases may still render below DPR 1, down to 0.5.

The canvas uses `frameloop="demand"`: scrolling, GSAP transitions, OrbitControls, resizing, and visibility changes explicitly request frames, while a static page consumes no render loop. Rendering is fully suspended while the tab is hidden. The scene avoids per-frame allocations, uses no postprocessing stack, uses one temporary Basis worker only for KTX2 variants, disposes it after loading, and provides both a PNG model fallback and a visible WebGL failure fallback.
