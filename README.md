# NORKA R35 — 3D Scroll Story

A Vite + React + strict TypeScript automotive storytelling experience using the real uploaded R35 GLB. The WebGL canvas is fixed full-screen while HTML sections scroll above it. One GSAP master timeline controls camera position, target, FOV, and section copy, so upward scrolling evaluates the exact same path in reverse.

The project is configured for **Yarn Classic 1.22.22 only**. `yarn.lock` is the single dependency lockfile.

## Requirements

- Node.js 20.19+ or 22.12+
- Yarn 1.22.22
- A browser with WebGL and hardware acceleration
- Non-commercial use because the supplied model is licensed CC BY-NC-SA 4.0

## Setup

```bash
yarn install --frozen-lockfile
yarn dev
```

Open the local URL printed by Vite.

## Production

```bash
yarn typecheck
yarn build
yarn preview
```

The production output is `dist/`. `vercel.json` is included; on Vercel use the Vite preset, build command `yarn build`, and output directory `dist`.

## Runtime assets

- `public/models/norka-r35-original.glb` — untouched reference copy of the uploaded GLB.
- `public/models/norka-r35-desktop.glb` — quality-first web model with exact float32 normals/tangents and 4096px reference maps embedded.
- `public/models/norka-r35-mobile.glb` — phone model with identical geometry and 2048px paint/carbon/glass maps to reduce decoded texture memory.
- `public/textures/norka-*-reference.png` and `norka-carbon-*.png` — optimizer inputs for the exact paint, glass, carbon albedo, and carbon normal maps used by the [vecarz Sketchfab edition](https://sketchfab.com/3d-models/norka-varis-r35-wwwvecarzcom-c4c42f9ef40644399f301a910184b241).
- `public/hdr/automotive-studio.hdr` — local procedural studio HDR generated by `scripts/generate-studio-hdr.py`.

Each generated web GLB is self-contained: the four reference maps are embedded once and reused by the runtime material restoration, avoiding duplicate image decoding. Run `yarn inspect:model`, `yarn inspect:model:mobile`, or `yarn inspect:model:original` to inspect each artifact. The visible footer attribution is read from `asset.extras` at runtime, with a defensive fallback only when metadata is absent; it also credits the vecarz material presentation.

Without an override, runtime selects `mobile` for phone-sized/coarse landscape viewports and `desktop` elsewhere. Set `VITE_MODEL_VARIANT=original`, `desktop`, or `mobile` only when forcing a comparison build.

## Scroll and camera architecture

`src/hooks/useScrollStory.ts` creates a single master ScrollTrigger in normal-motion mode. Its scrubbed timeline is the only writer to the story rig. Camera position uses two non-overlapping legs per section to form the orbit/raise/descend/pull-back path. Target and FOV span the complete section transition. Reverse scroll therefore evaluates the same timeline backward, with no separate camera animation on `onLeaveBack`.

`src/three/CameraRig.tsx` copies the mutable rig into the Three.js camera in `useFrame`; React state is not updated on every scroll frame. The projection matrix is updated only when FOV changes. Section text uses opacity and transforms.

For `prefers-reduced-motion`, the camera switches between static desktop/compact shots and copy uses short opacity changes.

## Refined camera compositions

The camera shots were visually re-tuned without modifying the model, normalization, model scale, CSS transforms, or timeline architecture. The exterior sequence now follows one direction through Hero → Performance → Aerodynamics → a straight-on Rear Signature → rear-left Precision, then enters the existing detailed cabin sequence. A later coverage pass also widens the Digital Cluster view to include the center display and routes the exterior-to-cockpit move through the rear cabin instead of through seat geometry. The responsive review viewports were:

- Desktop: `1440 × 900`
- Tablet landscape: `1024 × 768`
- Tablet portrait: `768 × 1024`
- Mobile: `390 × 844`
- Small mobile: `320 × 568`
- Mobile landscape: `667 × 375` and `844 × 390`

Desktop views use `desktopShots`; phones and tablets in portrait up to 1024px use `mobileShots` as the compact composition. Short landscape screens use `landscapeShots`, which inherits the desktop set and overrides the direct-rear and rear-left-detail compositions so the car remains prominent beside the copy. Touch phones retain mobile DPR/shadow limits after rotation. A portrait tablet keeps the desktop render quality and zoom limits while using safer portrait framing. The before/after coordinates, measured framing notes, and constraints check are recorded in [`CAMERA_TUNING_REPORT.md`](./CAMERA_TUNING_REPORT.md).

## Explore mode

The final CTA disables story ScrollTriggers without resetting their visual state, locks the page at its current scroll offset, eases to the final Explore composition, then enables OrbitControls. Pan is disabled, zoom is bounded, and polar limits prevent moving below the floor. Drag/touch, wheel/pinch, arrow keys, and `+`/`-` are supported. Exit disables controls, restores the final story shot, restores the same scroll offset, and re-enables ScrollTriggers without returning to the top.

## Development camera HUD

In development, press **D** to toggle the HUD. It shows camera position, target, FOV, current section, and master progress. **Copy shot** copies a valid `cameraShots.ts` object entry. The module is behind `import.meta.env.DEV` and is excluded from production output.

To show ScrollTrigger markers during development:

```bash
VITE_SCROLL_MARKERS=true yarn dev
```

## Camera tuning workflow

1. Run `yarn dev` and wait for the real GLB to reach 100%.
2. Press **D** to show the camera HUD.
3. Scroll to a section and inspect the actual composition.
4. Adjust the matching entry inside `desktopShots` or `mobileShots`; use `landscapeShots` only for the short-landscape override in `src/three/cameraShots.ts`.
5. Use **Copy shot** to capture position, target, and FOV.
6. Test both down-scroll and up-scroll. Target must continue tweening for the full section transition; do not introduce independent camera tweens.
7. Test portrait framing separately. Mobile shots are authored independently, not obtained by multiplying desktop coordinates.

## Model color restoration and compression

The original GLB is never overwritten. The downloaded model contains the complete geometry, UV sets, nodes, and 64 material slots, but the linked vecarz edition applies its dark carbon, black cabin, wheel, glass, and PBR settings inside the Sketchfab viewer. Those overrides are not stored in the downloadable GLB, which is why the untreated model renders large exterior and interior areas white or gray.

At runtime, `src/three/materialAdjustments.ts` restores the linked edition by exact material name. It applies the recovered linear color factors, metalness, roughness, opacity, clearcoat, alpha cutouts, UV channels, and up to 8× anisotropic texture sampling; it reuses the four maps embedded in the selected GLB and disables the three white emissive overrides that were not enabled in the reference presentation. Geometry, proportions, material slots, and camera normalization are unchanged.

The optimizer embeds the checked-in paint, carbon albedo, carbon normal, and glass maps into named material slots before compression. The runtime therefore does not download or decode a second copy of those textures.

The pipeline never simplifies, welds, joins, or removes model geometry. Position and in-range UV data use conservative 16-bit quantization, while every normal and tangent remains the original float32 data for smooth reflections. Desktop keeps the 4096px reference maps byte-for-byte; mobile uses high-quality Lanczos resampling to 2048px for only those three oversized color maps, then stores them as lossless PNG. All other embedded textures retain their original bytes and dimensions.

Optimizer dependencies are pinned in `devDependencies`:

```bash
yarn install --frozen-lockfile
yarn optimize:model
```

Default outputs:

- `public/models/norka-r35-desktop.glb`
- `public/models/norka-r35-mobile.glb`

Use `yarn optimize:model:desktop` or `yarn optimize:model:mobile` to rebuild one variant only. The runtime `useGLTF(..., true, true)` path includes Drei's Meshopt decoder.

The checked-in HDR is ready to use. To regenerate it, install Python 3 with NumPy and OpenCV, then run:

```bash
python -m pip install numpy opencv-python-headless
python scripts/generate-studio-hdr.py
```

## Performance and resilience

DPR is capped at 2 on desktop/tablet, 1.5 on mobile, and 1.25 on low-end mobile. Antialiasing remains enabled on every profile; shadows alone are reduced on low-end devices. Rendering pauses while the tab is hidden. The scene avoids per-frame allocations, uses no heavy postprocessing on mobile, and provides a visible WebGL fallback.
